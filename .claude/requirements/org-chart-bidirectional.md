# WI-190 조직도 ↔ 직원관리 양방향 CRUD + 후속 수정

## WI 번호: WI-190

## 요구사항
1. PR#229 후속 수정 재적용 — serviceKey 수정, 고용노동부 필터, 장려금/지원금 필터, 전체삭제, 원본 링크, 직원 기본정보(생년월일/성별/장애여부)
2. Guardian 에이전트 지침 수정 — 데이터 흐름 추적 체크리스트 추가
3. 직책 라벨 수정 — EmployeeDetailDrawer "팀장" → "직책"
4. 조직도→직원 연결 — 부서 클릭 시 소속 직원 목록 표시 + 직원 상세 드로어
5. 조직도→매니저 설정 — 부서의 매니저(직책) 지정 UI
6. 조직도→하위부서 생성 — 특정 부서 아래 하위 부서 생성 UI (parentId)
7. 직원관리→조직도 연결 — 직원 상세에서 소속 부서 클릭 시 조직도 이동

## 검증 기준
- 조직도에서 부서 클릭 → 소속 직원 목록 표시
- 조직도에서 매니저 지정/변경 가능
- 조직도에서 하위 부서 생성 가능
- 직원 상세에서 부서명 클릭 → 조직도 해당 부서로 이동
- 기존 CRUD 흐름 유지 (Admin 생성/수정, Employee 읽기만)
- 데이터 흐름: 조직도 매니저 설정 → 직원관리 직책 표시 연동
